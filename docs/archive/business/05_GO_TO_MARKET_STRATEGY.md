## 1. ГЛАВНЫЙ ПРИНЦИП

> **Мы строим маркетплейс. У маркетплейса одна проблема: «курица и яйцо» — покупатели не придут без продавцов, а продавцы не придут без покупателей.**

**Наше решение:** сначала продавцы → потом покупатели → потом маховик.

```
Фаза 1: Продавцы         →  Фаза 2: Покупатели    →  Фаза 3: Маховик
«Подключайся бесплатно»      «Найди проверенный       «Растёт само —
(месяц 1–3)                   инструмент»              каждая сделка
                               (месяц 3–6)              усиливает рост»
                                                        (месяц 6+)
```

**Почему сначала продавцы?**

- Им нечего терять — размещение бесплатное
- Каждый продавец приведёт 5–10 покупателей из своей аудитории
- Заполненный каталог привлекает покупателей сам по себе

**Воронка продаж:**

```

    ↓ 25% зарегистрировались
```

### Фаза 3: Маховик роста (Сентябрь–Декабрь 2026) — «Само растёт»

**Цель:** самоподдерживающийся рост без пропорционального увеличения расходов.

#### Контур 1: Продавец → Покупатель

```
Продавец публикует продукт → Рассказывает своей аудитории → 
Покупатели приходят → Сделки → Продавец зарабатывает → 
Публикует ещё → Рассказывает ещё → ...
```

#### Контур 2: Доверие как самоусиление

```
Больше сделок → Точнее рейтинги доверия →
Покупатели больше доверяют → Больше сделок →
Данные накапливаются → Конкурентам повторить невозможно
```

#### Контур 3: AI находит AI

```
AI-система A находит AI-систему B через Agora →
Сделка → Рейтинг обновляется →
AI-система A рекомендует Agora другим AI-системам →
Сеть растёт → Больше AI-систем подключаются
```
